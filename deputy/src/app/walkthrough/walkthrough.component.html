Part One

Question 1 HTML

I created a non-functional login screen, mirrored as closely as possible to the mockup shown in the instructions.

Question 2 JavaScript & jQuery

I re-created the people page, and added a few Material Design touches. The filter is fully functional, allowing searches by User Id, User DisplayName, Location Id, Location Name, Role Id, and Role Name. The filter works by itself or with the dropdown that restricts the search category by specific search entities.

The data is retrieved using a NodeJS backend that I created (server.js). I utilized an Interface design pattern. Each data entity (UserData, RoleData, LocationData, etc) implements the Data interface since they have one property in common, the Id. This allowed me to make my rest calls generic enough for resuse.

Question 3 Shift Overlap Detection

I mocked data for several employees and then created a table to display them. Adding and deleting of schedules is functional as well as form validation which checks for existing user and department before assigning a schedule.

Question 4 Users Hierarchy

I used expansion panels to assist in grouping the users by role, and tables to aid in displaying the employee data. I also added a checkbox to restrict the results to the location of the user.
